## Arch Linux Enthusiast since 2008
**Arch Linux** isn't just a distro for me; it's been my daily driver since 2008. No other backup OS, no dual boot. My journey with Linux begin 25+ years ago with a distro I can barely remember (and didn't particularly like), but everything changed in 2008.

**My Evolution:**
1. **The Beginning:** I started with **Ubuntu** on a modest, Atom-powered Asus Eee PC 1201N. It did the job, but I quickly craved something lighter.
2. **The Transition:** I moved to **CrunchBang** (Ubuntu + Openbox), but I didn't like the slow release cycles—I wanted the latest features and security patches. Having the latest browser was a big deal back then; sometimes it meant better performance and being able to play YouTube videos without having to mess with extra drivers.
3. **The Discovery:** That led me to **ArchBang** (Arch Linux + Openbox), and that’s where my obsession with Arch began.
4. **The Long Run:** From 2009 until recently, my daily driver was a minimal from scratch **Arch + Openbox** setup, paired with lightweight XFCE utilities.

**Stability & Reliability:**
In nearly 18 years, I’ve faced only one real issue where I couldn't boot. It was a kernel regression years ago that required a quick fix via an Arch Live USB. For a rolling release distro, that level of stability is mind-blowing.

> **[!TIP]**  
> Pro Tip: I always install a second `linux-lts` kernel alongside the main `linux` one.  
> If a new update ever breaks things, I can just boot into the LTS version. Interestingly, I haven't actually needed to use this fallback yet!

**My Philosophy:**
In 2023, I migrated from my long-standing **Openbox/XFCE** setup to **Hyprland (Wayland)**. I still stick to the core principles:
1. **Manual Installs**: Built from scratch, no automated scripts.
2. **Minimalism**: Minimal applications. Only the packages I actually need.
3. **Stay Current**: Regular updates, bleeding-edge software.

Total control over the system with latest updates.
